# Qualité - Accessibilité (RGAA 4.1)

## Cible

- **AA sur 100% des pages** (cible légale).
- **AAA sur 5 parcours critiques** : inscription, connexion, création de post, lecture du feed, envoi de message chat.

## Critères automatisés en CI

Outils :
- `axe-core` via Playwright sur les 10 parcours principaux.
- `pa11y-ci` sur snapshots HTML statiques.
- `lighthouse-ci` avec seuil score accessibilité ≥ 95.
- `eslint-plugin-jsx-a11y` au commit.

Quality gate CI : tout PR qui dégrade un de ces scores est bloqué.

## Préférences utilisateur persistées

Stockées en table `user_preferences`, appliquées via classes CSS et CSS variables :
- Taille de police (sm, md, lg, xl).
- Espacement (compact, normal, large).
- Thème (light, dark, auto).
- Contraste élevé (mode AAA).
- Réduction de mouvement.
- Mode sobriété (texte uniquement, pas d'images, pas de WS - aussi argument eco).

## Parcours critiques RGAA AAA - check-list

Pour chaque parcours, valider :

- [ ] Navigable au clavier seul (Tab, Shift+Tab, Enter, Esc, flèches).
- [ ] Lecteur d'écran annonce les changements d'état (NVDA + Orca testés).
- [ ] Aucun piège au focus (sauf modales, où le focus reste piégé volontairement et Esc ferme).
- [ ] Messages d'erreur annoncés via `aria-live="polite"` ou `role="alert"`.
- [ ] Contraste 7:1 (AAA) sur le contenu, 4.5:1 sur les éléments d'interface décoratifs.
- [ ] Texte alternatif pour images informatives.
- [ ] Cibles tactiles ≥ 44x44px (critère 2.5.5 WCAG AAA).

## Mode sobriété (cumule a11y et eco)

Page `/parametres/affichage` permet d'activer "Mode sobriété" :
- Pas d'images affichées (placeholders `[image: <alt>]`).
- Pas de WebSocket (polling 30s à la place ou notification banner only).
- CSS minimal, pas d'animations.
- Idéal pour : malvoyants, connexions lentes, économie de batterie.

## Déclaration d'accessibilité

Page `/legal/accessibilite` au format DINUM :
- Niveau revendiqué.
- Date du dernier audit.
- Liste des contenus non conformes (le cas échéant).
- Contact pour signaler un problème.
- Voies de recours.

Modèle : https://accessibilite.numerique.gouv.fr/ressources/declaration/

## Audit final

Une session manuelle de 2h en S4 :
- NVDA sur Windows OU Orca sur Linux (à défaut).
- Test des 5 parcours AAA en mode lecteur d'écran.
- Documentation des écarts dans `docs/audits/a11y-vX.Y.Z.md`.
